# 🚀 Интеграция Supabase с Bolt - Полное руководство

## 📋 **Обзор подключения**

### 1. **Конфигурация базы данных**
- **URL проекта**: Использует переменную `NEXT_PUBLIC_SUPABASE_URL`
- **API ключ**: Использует переменную `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- **Таймаут подключения**: Встроенная обработка клиента Supabase
- **Обработка ошибок**: Комплексные механизмы резервного копирования

### 2. **Структура таблиц базы данных**

**Таблица `exchange_rates`:**
```sql
- id (uuid, первичный ключ)
- source (text) - 'kenig', 'bestchange', 'energo'
- usdt_sell_rate (decimal 10,4) - Курс продажи USDT
- usdt_buy_rate (decimal 10,4) - Курс покупки USDT
- created_at (timestamptz) - Время создания
- is_active (boolean) - Активность записи
- metadata (jsonb) - Дополнительные данные
```

**Безопасность на уровне строк (RLS):**
- ✅ Публичный доступ на чтение курсов
- ✅ Доступ на запись для аутентифицированных пользователей
- ✅ Оптимизированные индексы для производительности

### 3. **Паттерны взаимодействия**

**Операции чтения:**
- `getLatestRates()` - Получение текущих активных курсов
- `getRatesHistory()` - Получение исторических данных с пагинацией
- Обновление курсов в реальном времени каждые 30 секунд

**Операции записи:**
- `saveRates()` - Вставка новых курсов и деактивация старых
- Автоматическое управление временными метками
- Поддержка метаданных для дополнительной информации

**Паттерны запросов:**
- Фильтрация по источникам
- Фильтрация по статусу активности/неактивности
- Сортировка по времени
- Эффективная индексация для быстрых запросов

## 📊 **Примеры использования**

```typescript

const rates = await getLatestRates();
```

## 🛠️ **Инструкции по настройке**

### Шаг 1: Настройка Supabase проекта
1. Создайте новый проект в [Supabase Dashboard](https://app.supabase.com)
2. Перейдите в Settings → API
3. Скопируйте "Project URL" и "anon public" ключ

### Шаг 2: Настройка переменных окружения
Создайте файл `.env.local` в корне проекта:
```bash
NEXT_PUBLIC_SUPABASE_URL=https://ваш-проект.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=ваш-anon-ключ
```

### Шаг 3: Выполнение миграции
1. Откройте Supabase Dashboard
2. Перейдите в SQL Editor
3. Выполните SQL из файла `supabase/migrations/20250617163540_smooth_feather.sql`

### Шаг 4: Тестирование
1. Перезапустите сервер разработки: `npm run dev`
2. Откройте `/debug-supabase` для проверки всех компонентов
3. Откройте `/test-supabase-rates` для тестирования функций
